# Customer Shopping Behavior Analysis
**Tools:** Python (pandas) · PostgreSQL (Supabase) · Power BI  
**Skills:** Data Cleaning · SQL Analysis · Dashboard Design · Customer Segmentation

---

## Project Overview

This project analyzes customer shopping behavior using a dataset of 3,900 customers across four product categories — Clothing, Accessories, Footwear, and Outerwear.

The goal was to answer real business questions around revenue patterns, customer segmentation, discount effectiveness, and subscription performance using end-to-end data analysis: from raw data cleaning in Python to SQL querying on a cloud database to an interactive Power BI dashboard.

---

## Business Questions Answered

1. What is the total revenue generated by male vs. female customers?
2. Which customers used a discount but still spent above average?
3. Which products have the highest average review ratings?
4. Does shipping type affect purchase amount?
5. Do subscribed customers actually spend more than non-subscribers?
6. Which products are most heavily discounted?
7. How is the customer base distributed across New, Returning, and Loyal segments?
8. What are the top 3 most purchased products in each category?
9. Are repeat buyers more likely to subscribe?
10. Which age group contributes the most revenue?

---

## Key Findings

- **Male customers generate 68% of total revenue** ($157,890 vs $75,191 for female), despite both genders being present in the dataset.
- **Subscription program underperforms** — subscribers and non-subscribers spend nearly the same per transaction (~$59.49 vs ~$59.87), and 72% of repeat buyers have not subscribed.
- **Discount rates are critically high** on top products — Hat (50%), Sneakers (49.66%), Coat (49.07%) — raising margin concerns, especially since Hat also ranks among the highest-rated products.
- **80% of customers are Loyal** (11+ previous purchases), but only 83 new customers exist in the dataset — a potential acquisition gap.
- **Revenue is evenly distributed across all age groups** ($55K–$62K), indicating balanced demographic reach with no over-reliance on a single segment.

## How to Run

1. Clone the repo
2. Open `notebooks/data_cleaning.ipynb` in Google Colab
3. The notebook connects to a Supabase PostgreSQL database *(connection string stored as environment variable — not included for security)*
4. SQL queries are standalone in `Sql Query for Business Questions.pdf` — run them against any PostgreSQL instance with the same schema
5. Power BI file connects to the same database; use the dashboard screenshot for reference

## Dataset

- **Source:** [Customer Shopping Trends Dataset – Kaggle](https://www.kaggle.com/datasets/iamsouravbanerjee/customer-shopping-trends-dataset)
